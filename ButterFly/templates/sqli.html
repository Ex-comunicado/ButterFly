SQL Injection

Every WebApp has a back-end database to store information. For the WebApp to perform operations such as signing up, logging in, showing stored information to user the back-end database comes into use. When the user is allowed to provide input and it makes the Web-App interact with that back-end database. If proper measures are not taken, the user’s input can affect the back-end database because a malicious user can modify the query running to either gain access, gain sensitive information, or modify/delete values in the database. SQL Injection is the attack that enables an attacker to inject SQL query into the code, modifying the operation of the intended query. One type of SQL Injection vulnerabilities is Blind SQLi – This type of vulnerability arises when the application shows generic error messages that do not indicate an SQLi vulnerability but the vulnerability exists nevertheless and can be exploited by asking the database true/false questions and determine the answer based on the application’s response. The types of SQLi attack are: Time-based and Boolean-based.
