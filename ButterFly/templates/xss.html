Cross Site Scripting (XSS)

XSS vulnerability occurs whenever an application includes user supplied data into a new web page without proper escaping or validation or updates the existing web page with the input. XSS allows attackers to execute scripts on a victim’s browser taking over their session, deface websites or redirect the user to a malicious site. There are 3 types of XSS attacks:
  • Reflected XSS: The malicious script (JavaScript code) comes from the current HTTP request and the application includes that data in the immediate response in an unsafe manner.
  • Stored XSS: The malicious script in an HTTP request is included in further HTTP responses in an unsafe way. The script is stored in the application’s database and is therefore run every time that particular resource is accessed. For e.g., a comment on a website.
  • DOM based XSS.

Preventing XSS

  • Filter data on arrival.
  • Encode data on output.
  • Use appropriate response headers. To prevent XSS in HTTP responses that aren't intended to contain any HTML or JavaScript, you can use the Content-Type and X-Content-Type-Options headers to ensure that browsers interpret the responses in the way you intend.
