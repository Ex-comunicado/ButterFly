OS Command Injection

Command Injection is a vulnerability that allows an attacker to run arbitrary commands on the host OS of the web application and essentially compromise the whole system. Any parameter that accepts input and runs commands on the server can be vulnerable to this type of attack if the input is not sanitized appropriately. This type of vulnerability can be checked for by separating intended and malicious commands by shell such as &, &&, |, ||, ;, newline character. Malicious commands may differ based on what OS resides in the back-end.

Preventing OS Command Injection

  • Don't use OS commands where user can supply input.
  • If previous point unavoidable, sanitize user input by implementing a whitelist of permitted values.
  • Reduce the acceptable input character-set to a very narrow value, such as only alphanumeric characters so that any meta-characters or whitespaces are not accepted as input.
